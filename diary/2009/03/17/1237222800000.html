JJBug、JJug共催 - JBoss ナイトセミナ第３回へ行ってきました

標記の通り、JBossナイトセミナ第3回に行ってきました。
・日本 Java ユーザグループ - JBoss ナイトセミナ 第3回

私も１セッション担当させていただきました。内容は JBoss プロダクト群のなかでもややマイナーな JBoss Rules(=Drools)。
JBoss Rules(=Drools)は Pure Java のビジネスルールエンジンです。もちろん JBoss なのでオープンソース。無償利用可能。
ビジネスルールエンジンと聞くとなんだか仰々しくて手を出しにくい印象がありますが、比較的簡単に適用でき、応用範囲の広い製品であることを少しは説明できたのではないかとおもいます。

とりとめもなく52ページもあるスライドですが、SlideShare で公開しておきます。

JBoss Rules
View more presentations from yusukey.


デモの内容はログを監視して、メッセージの内容に応じて標準出力にダンプ、メール送信、Twitterをアップデート、Growlでデスクトップに通知といったアクションをダイナミックに設定できるというものでした。
Twitter をアップデートするのには失敗しました。これはTwitter4Jのバグではなく、同じメッセージを連続で投稿できないという Twitter の仕様によるものでした。思いつきで事前検証もなくデモを失敗することは良くありますが、逆に直前に検証を行ったことで失敗してしまったのは初めてです。

先のエントリで公開したQBはこのデモ用に作ったアプリケーションです。やっつけで作り、セッションに間に合うよう無理矢理パッケージしました。まだまだ完成度が低いですが少しずつ膨らませて行きたいと思っています。


ご意見、ご感想などございましたらコメントやトラックバックで頂ければ幸いです。

最後に、スライドで紹介している関連リンクを張っておきます。

I experienced this issue yesterday and after doing some debugging, Below is my findings and solution:
FYI: I am using version 2.1.3

UserJSONImpl.init(JSONObject json){
            ......
            createdAt = getDate(json.getString("created_at"), "EEE MMM dd HH:mm:ss z yyyy"); //throws an error when "created_at" does not exist when "status = new StatusJSONImpl(statusJSON);" is called in UserJSONImpl.init(JSONObject json)
            favouritesCount = json.getInt("favourites_count"); //throws an error when "favourites_count" does not exist when "status = new StatusJSONImpl(statusJSON);" is called in UserJSONImpl.init(JSONObject json)
            .....
}

My working Solution:
UserJSONImpl.init(JSONObject json){
            ......
            createdAt = getRawString("created_at", json) == null?null:getDate(json.getString("created_at"), "EEE MMM dd HH:mm:ss z yyyy");
            favouritesCount = getRawString("created_at", json) == null?0:json.getInt("favourites_count");
            .....
}

Description

I like to have the source code for a library I'm using.

Right now, it looks like the only way to download source that matches the downloaded jar is to try to find the point in git when the jar was released.

Instead, perhaps add a srcjar target in twitter4j-core/build.xml that is something like

    <target name="srcjar" if="version">
        <jar jarfile="${version}-src.jar" basedir="${src}"/>
    </target>

Just a request.

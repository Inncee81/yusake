null pointer exception getting user detail on users with no statuses

Created: 13/Mar/09 10:23 AM   Updated: 23/Mar/09 12:29 AM
Component/s: Core
Affects Version/s: None
Fix Version/s: 1.1.8


 Description  « Hide
http://groups.google.com/group/twitter4j/browse_thread/thread/86a2e9386e739dcc

If you call getUserDetail() against an account with no statuses, you
will get a null pointer exception because there are no status messages
for the account. ("bustedtees" is an example that will fail.)

Easy fix should be just to do a null check after 'Element status
= ...' like so:

   public UserWithStatus(Element elem, Twitter twitter) throws
TwitterException {
       super(elem, twitter);
       profileBackgroundColor = getChildText
("profile_background_color", elem);
       profileTextColor = getChildText("profile_text_color", elem);
       profileLinkColor = getChildText("profile_link_color", elem);
       profileSidebarFillColor = getChildText
("profile_sidebar_fill_color", elem);
       profileSidebarBorderColor = getChildText
("profile_sidebar_border_color", elem);
       friendsCount = getChildInt("friends_count", elem);
       favouritesCount = getChildInt("favourites_count", elem);
       statusesCount = getChildInt("statuses_count", elem);
       if (!isProtected()) {
           Element status = (Element) elem.getElementsByTagName
("status").item(0);
           if (status != null) { // NEW LINE
              statusCreatedAt = getChildDate("created_at", status);
              statusId = Long.valueOf(status.getElementsByTagName
("id").item(0).getTextContent());
              statusText = getChildText("text", status);
              statusSource = getChildText("source", status);
              statusTruncated = getChildBoolean("truncated", status);
              statusInReplyToStatusId = getChildLong
("in_reply_to_status_id", status);
              statusInReplyToUserId = getChildInt
("in_reply_to_user_id", status);
              statusFavorited = getChildBoolean("favorited", status);
              statusInReplyToScreenName = getChildText
("in_reply_to_screen_name", status);
           } // NEW LINE
       }
   }
